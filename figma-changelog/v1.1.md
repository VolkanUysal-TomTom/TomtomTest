# Figma Changelog — v1.1

**Release date:** 2026-04-15
**Figma library:** TomTom Navigation DS — Base Tokens & Components

---

## Summary

This release updates the Figma library to align with the v1.1 token manifest. Changes affect the guidance NIP, EV charging indicators, panel safe-area spacing, and the deprecation of a legacy warning colour. All changes are non-breaking for existing component instances unless a designer has overridden the renamed variable locally.

---

## Layout Changes

### 1. Next Instruction Panel (NIP) — accent variable renamed

- **Affected frames:** All NIP variants in `Guidance / NIP / [size]`
- **What changed:** The variable `tt_sys_color_nip_accent` has been renamed to `tt_sys_color_guidance_accent` across every NIP frame, the Lane Guidance overlay, and the Speed Shield component.
- **Visual impact:** None — the resolved colour value (`#1988cf`) is unchanged.
- **Action required for file detachers:** If you detached any NIP component and are referencing the old variable name by string, update to the new name. Attached instances will auto-resolve.

---

### 2. EV Charging Indicator — new availability colour

- **Affected frames:** `Map Pins / EV Charging / Available`, `EV Panel / Connector List / Availability Pill`
- **What changed:** A new Figma variable `tt_sys_color_evCharging_available` (resolved: `#03bf95`, Mint 300) has been applied to the "available" connector state. Previously this state was using the generic `tt_glb_color_green_200` global token directly.
- **Visual impact:** Slight hue shift — the new mint tone is more distinct from the route-line green (`tt_glb_color_green_300`) and improves colour separation on the map layer.
- **Action required:** Review any custom EV availability screens and apply the new semantic variable. Do not use the global `tt_glb_color_green_*` tokens directly on EV states going forward.

---

### 3. Floating & Docked Panels — bottom safe-area spacing

- **Affected frames:** `Panels / Floating / [all sizes]`, `Panels / Docked Bottom / [all sizes]`, `Panels / Bottom Docked Not Collapsible`
- **What changed:** A new auto-layout padding variable `tt_sys_spacing_panel_safeArea_bottom` (48 dp) has been applied to the bottom padding of all floating and bottom-docked panel frames. Previously this padding was hard-coded to `32` and was inconsistent across panel variants.
- **Visual impact:** Panels now have 48 dp of bottom clearance instead of 32 dp. This is intentional to accommodate the Android gesture navigation bar and iOS home indicator.
- **Action required:** If you are mocking device frames with a gesture bar overlay, verify that panel content does not overlap the bar. Adjust mockup artboard heights if needed — panels will be taller by 16 dp.

---

### 4. Route Message Banner — legacy warning colour deprecated

- **Affected frames:** `Notifications / Route Message / Warning`
- **What changed:** The fill on the warning-state Route Message banner has been migrated from the deprecated `tt_cmp_color_routeMessage_warningLegacy` to the system-level `tt_sys_color_feedback_warning`. The resolved value remains `#fc823a` (Orange 200) for now, but this will change when the feedback palette is updated in a future release.
- **Visual impact:** No immediate change in this release.
- **Action required:** Designers should remove any local override to `tt_cmp_color_routeMessage_warningLegacy` and let the component pick up `tt_sys_color_feedback_warning` from the library. The legacy variable will be removed in v2.0.

## Variable Mode Updates

| Mode set | Change |
|---|---|
| `Light` | Added `tt_sys_color_evCharging_available` and `tt_sys_color_guidance_accent` |
| `Dark` | Added `tt_sys_color_evCharging_available` (resolved: `#027c61`, Mint 400) and `tt_sys_color_guidance_accent` (resolved: `#53A5DD`, Blue 350) |
| `Large` screen | Added `tt_sys_spacing_panel_safeArea_bottom` (value: 48) |
| `Small` screen | Added `tt_sys_spacing_panel_safeArea_bottom` (value: 40, overridden for smaller gesture bar zone) |
